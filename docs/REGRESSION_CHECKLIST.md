# 回归测试清单（Bug 修复验证）

在每次发版或合并本批修复后，按顺序勾选验证。环境：本地 `npm run dev` + 如需联调则 `backend` 已启动。

## P0 / P1

- [ ] **导航定时器**：对局结束或放弃进入结果页时，快速重复触发结束流程，结果页不应出现多次跳转或异常路由栈。
- [ ] **存档 answerCache**：制造一次网络/API 失败后刷新页面，校验同一问题仍复用（或显示）失败兜底文案，且不因非法缓存整页失效。
- [ ] **联调 URL**：`VITE_API_BASE` 指向自有后端时，`/api/chat` 请求发到该基址（浏览器 Network 中确认），而非硬编码相对路径错误。

## P2

- [ ] **错误体解析**：后端返回非 200 且 body 含 `message` 或 `error.message` 时，前端 Toast/Notice 能带出可读说明（非仅有 HTTP 码）。
- [ ] **结果页推理记录**：对局中触发阶段提示与手动提示后进入结果页，推理记录中应出现「（阶段提示）」「（手动提示）」行及对应气泡全文。
- [ ] **存档写入失败**：在模拟存储失败（如 DevTools Application → 禁用 sessionStorage 或灌满配额）时，应出现一次「本地存档写入失败」类提示，对局可继续玩。

## P3

- [ ] **X-Request-Id**：Network 中 `POST /api/chat`、`/api/hint` 或 `/api/manual-hint`、直连模型时的请求头含 `X-Request-Id`，且每次请求不同。
- [ ] **局域网 preview**：用 `192.168.x.x` 或 `10.x.x.x` 访问前端时，OpenAI 仍走本地代理路径（与 `localhost` 行为一致），无 CORS 直连失败。

## 回归烟雾

- [ ] `web`：`npm run build`、`npm test` 通过。
- [ ] 新建一局 → 提问 → 通关/失败/放弃 三条路径各走通一页。
- [ ] Reveal 页汤底与后端/落库一致，无前端二次生成。
