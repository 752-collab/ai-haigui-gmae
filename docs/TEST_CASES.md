# AI 海龟汤 — 详细测试用例（可执行清单）

**前缀说明**  
- **测试类型**：功能（F） / 兼容性（C） / 异常（E）  
- **优先级**：P0 阻塞发布 / P1 重要 / P2 一般  

---

## 一、游戏大厅（Home）

| ID | 测试模块 | 测试步骤 | 预期结果 | 类型 | 优先级 |
|----|----------|----------|----------|------|--------|
| H-01 | 大厅渲染 | 启动前端，浏览器访问 Vite Local URL | 出现标题「AI海龟汤」、玩法介绍区块、【玩法介绍】下至少一条列表项 | F | P0 |
| H-02 | 故事卡片 | 在大厅向下滚动 | 与 `stories` 数据一致的卡片网格；每张卡片含标题、难度标签 | F | P0 |
| H-03 | 卡片跳转 | 点击任一故事卡片 | 路由变为 `/game/{id}`；游戏页加载无白屏 | F | P0 |
| H-04 | 直连 URL | 地址栏输入 `/game/unknown-id` | 进入游戏后按产品逻辑处理未知 ID（不崩溃：空白/404/回退） | E | P1 |
| H-05 | 移动端布局 | DevTools 切换 iPhone/Android 宽度 | 网格为单列；文字可换行；无横向溢出 | C | P1 |
| H-06 | 弱网首屏 | Network Throttling: Slow 3G，刷新大厅 |  eventually 渲染完成或可接受超时提示；控制台无致命错误 | C | P2 |

## 二、游戏对局（Game）

| ID | 测试模块 | 测试步骤 | 预期结果 | 类型 | 优先级 |
|----|----------|----------|----------|------|--------|
| G-01 | 汤面展示 | 从大厅进入某一 `id` | 页头展示与 `stories` 一致的标题、难度、汤面正文 | F | P0 |
| G-02 | 发送消息 | 输入合法是非问句，点击发送 | 用户气泡出现；随后出现 AI 气泡（或加载态后 AI） | F | P0 |
| G-03 | AI 回答枚举 | 多轮提问 | AI 回复语义落在「是 / 否 / 无关」（及允许的 proximity/stageHint 结构） | F | P0 |
| G-04 | 判题与汤底 | 用已知汤底构造问题「与汤底一致」与「矛盾」 | 分别得到「是」与「否」（允许偶发模型不稳，需记录） | F | P1 |
| G-05 | 查看汤底 | 点击「查看汤底」或流程结束后进入结果页 | 跳转 `result` 路由；展示与当局一致的汤底/标题信息 | F | P0 |
| G-06 | 再来一局 | 从结果或大厅再次进入同一/不同故事 | 新会话或可恢复会话符合产品设计；无脏状态卡死 | F | P1 |
| G-07 | 手动提示 | 在 playing 状态点击「手动提示」最多 3 次 | 聊天区出现三档递进提示前缀；用尽后按钮不可用 | F | P1 |
| G-08 | 非法输入·纯数字 | 输入框输入 `1` 或 `99` 点发送 | 前端提示需用是非提问或点手动提示；**不**应出现无意义「是」 | E | P0 |
| G-09 | 重复点击发送 | AI 思考中快速连点发送 | 不应产生重复请求导致计数错乱；UI 禁用或与后端一致 | E | P0 |
| G-10 | 空消息 | 仅点发送，内容为空 | 提示不能为空；不发起判题请求 | E | P0 |
| G-11 | 超长输入 | 粘贴 5k+ 字符发送 | 前端截断/拒绝或后端 413；不崩溃 | E | P1 |
| G-12 | 边界·第 20 次问 | 模拟或真实提问至上限 | 符合 PRD：不可再问或后端拒绝；可揭晓 | F | P1 |
| G-13 | 刷新页面 | 对局中 F5 | sessionStorage 策略：恢复或合规重置；不无限 loading | C | P1 |
| G-14 | 后退 | 对局中浏览器返回 | 路由与状态可用；不泄露未授权汤底到 URL | C | P1 |

| ID | 测试模块 | 测试步骤 | 预期结果 | 类型 | 优先级 |
|----|----------|----------|----------|------|--------|

## 六、自动化与手动关系

- **Vitest**：单测前端纯函数、轻量组件与路由链接（不依赖真实 DeepSeek）。  
- **Jest + Supertest**：契约测试 HTTP 与可 Mock 的判题链路。  
- **手动 / E2E**（Playwright/Cypress 未包含）：覆盖大厅 → 对局 → 结果全链路、移动端视口。  

执行命令见各目录 `package.json` 中 `test` 脚本。
